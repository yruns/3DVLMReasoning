# 2026-03-25 踩坑记录：Florence-2 替换 RAM+GDINO 检测前端

## 背景

目标：用 Florence-2 替代 ConceptGraph 的 RAM+GroundingDINO+SAM 检测管线，消灭垃圾标签（`green`, `lead to`, `other item` 等），同时保持召回率。

## 坑 1：.venv 遮蔽 conda python — 最大的坑

### 症状
SAM 模型"加载失败"，但脚本没有报错，继续运行，生成的 mask 全是矩形（bbox 填充）。下游 3D mapping 质量极差，物体互相"渗透"。

### 排查过程
```
$ which python
/home/ysh/codecase/3DVLMReasoning/.venv/bin/python   # ← 问题在这里

$ python -c "import torch"
ModuleNotFoundError: No module named 'torch'
```

项目根目录的 `.venv`（uv 创建的）在 PATH 中优先级高于 conda env，即使执行了 `conda activate conceptgraph`，`python` 仍然解析到 `.venv/bin/python`——这个环境没有 torch/SAM/open_clip。

### 根因
代码里有 try/except 静默吞掉 ImportError：
```python
try:
    from segment_anything import SamPredictor, sam_model_registry
except ImportError:
    SamPredictor = None  # 静默降级，没有任何报错
```

之后的逻辑在 `SamPredictor is None` 时 fallback 到矩形 mask，用户完全感知不到。

### 修复
1. 删除项目 `.venv` 目录，统一使用 conda `conceptgraph` 环境（Linux）
2. 删除所有 try/except 静默 fallback，SAM 导入改为顶层 hard import：
   ```python
   from segment_anything import SamPredictor, sam_model_registry  # MUST succeed
   ```
3. SAM checkpoint 不存在时直接 `raise FileNotFoundError`

### 教训
**关键依赖（torch, SAM, CLIP）绝不能用 try/except 静默降级。** 宁可 crash 也不能静默产出垃圾数据。已写入 CLAUDE.md 作为项目规范。

## 坑 2：Florence-2 `<OD>` 召回率灾难性低

### 症状
`<OD>` 模式每帧只检测 3-5 个最显著物体。5 个场景总共只有 120 个物体（v1 有 229 个）。Scene 014（开放式客厅）只检测到 **7 个物体**。

### 尝试过但没用的方法
- `<DENSE_REGION_CAPTION>`：和 `<OD>` 返回完全相同的 bounding box，只是文本更丰富
- `<REGION_PROPOSAL>`：同上，框一样，且没有标签
- `<MORE_DETAILED_CAPTION>` → 提取名词 → grounding：caption 覆盖不全，提升有限

### 解决方案：手写词表 + `<CAPTION_TO_PHRASE_GROUNDING>`
给 Florence-2 一个广泛的室内物体词表，让它对照图片做 grounding：
```python
"chair. table. desk. sofa. couch. bed. bench. stool. ottoman. armchair."
```

单帧检测数从 3 个飙升到 **16 个**。

## 坑 3：词表过长导致标签拼接 bug

### 症状
把 ~90 个词一次性传给 Florence-2，最后一个检测的标签变成：
```
trash can. bin. basket. bag. backpack. shoe. clothing. towel. sink. faucet. toilet. bathtub. shower. dishwasher. countertop. nightstand. dresser. wardrobe. closet
```
Florence-2 的 token 生成到极限后，把剩余词汇全拼成了一个 label。

### 修复
将词表拆成 9 个 batch，每批 ~10 个词，分别调用 grounding：
```python
_INDOOR_VOCAB_BATCHES = [
    "chair. table. desk. sofa. couch. bed. bench. stool. ottoman. armchair.",
    "cabinet. shelf. drawer. dresser. wardrobe. closet. nightstand. bookshelf. rack.",
    ...  # 共 9 批
]
```

### 代价
每帧从 1 次推理变成 10 次（1 OD + 9 grounding），速度从 ~1s/帧降到 ~4s/帧。

## 坑 4：NMS 去重不够 → 大量重复检测

### 症状
OD 检测到 `chair`，grounding 同时检测到 `sofa`/`couch`/`armchair`，指向同一个物体。去重后仍有语义重复（不同标签，相同框）。

### 修复
IoU-based NMS，阈值 0.7，优先保留较小/更具体的框：
```python
def _nms_merge(bboxes, labels, iou_thresh=0.7):
    # 按面积升序排列，小框优先保留
    ...
```

## 坑 5：词汇选择导致系统性误检

### 症状
- `pen` 在所有场景中匹配细长物体 → 18/314 = 5.7% 假阳性
- `column` 过度匹配门框/墙角 → 15/314 = 4.8%

### 尝试但没用的方法
用 CLIP image-text similarity 过滤：pen 的 CLIP 相似度（0.262）和正常检测（0.286）差距太小，无法有效区分。

### 修复
直接替换词汇：`pen` → `remote control`（更有辨识度），`column` → `pillar`（更具体）。

## 最终结果

| 指标 | RAM+GDINO (v1) | Florence-2 Hybrid |
|------|----------------|-------------------|
| 总物体 | 229 | **312 (+36%)** |
| 类别数 | 87 | **77** |
| 垃圾标签 | 3 (`green`, `lead to`, `other item`) | **1** (`light`) |
| 垃圾率 | 3.1% | **0.3%** |

## 核心设计决策

1. **两遍检测**：`<OD>`（高精度）+ 词表 grounding（高召回），NMS 去重
2. **词表分批**：9 组 × ~10 词，避免 token 拼接 bug
3. **stride=5**：120 帧/场景，在速度和覆盖率之间取平衡
4. **no-fallback 规范**：所有关键依赖 hard fail，禁止静默降级
