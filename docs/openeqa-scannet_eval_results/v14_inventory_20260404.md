# v14 — Enrichment Inventory in System Prompt

**Date:** 2026-04-04
**Branch:** `feat/v11-evidence-seeking-fixes`
**Eval:** 1050Q full OpenEQA ScanNet
**Data:** `tmp/openeqa_eval_v14_full/`

## What Changed

**Single change:** Inject enrichment object inventory (category + description) directly into Stage 2 system prompt via `_format_scene_inventory()`.

Previously, the agent had to call `retrieve_object_context()` tool to access enrichment data. Analysis showed 51% of failures had GT object in enrichment but the agent never called the tool. Now every object's category and description (truncated to 120 chars) is visible upfront.

### Code Changes

- `runtime/base.py`: Added `_format_scene_inventory()` static method, modified `build_system_prompt()` to accept and inject `object_context`
- `runtime/deepagents_agent.py`: Pass `bundle.object_context` to `build_system_prompt()`
- `stage2_deep_agent.py`: Same caller update

### Key Observations

1. **Attribute Recognition +3.4** — largest category gain. Enrichment color/description info helps VLM identify correct colors and object attributes
2. **Spatial Understanding +2.6** — enrichment location/nearby_objects info helps spatial reasoning
3. **Functional Reasoning +2.6** — enrichment usability/affordance descriptions help functional questions
4. **Localization -1.6** — slight regression, possibly from longer prompt diluting attention
5. **Tool calls slightly decreased** (1.36→1.33/Q) — agent relies more on prompt inventory, less on tool calls
6. **Score=1 reduced by 17** — fewer completely wrong answers
